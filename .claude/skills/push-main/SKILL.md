# mainへのpush

このリポジトリはブランチを切らず、**直接mainブランチにpushする運用**。レビュー役がいないため、確認は最小限にしつつ、事故を防ぐチェックだけ行う。

## 手順

1. `git status` と `git diff --stat` で、今回の変更が**呼びかけられたサービスフォルダだけ**に閉じているか確認する
   - もし関係ない別のサービスフォルダ(design/mvp/investor等、自分が今触っていないはずのフォルダ)にまで変更が及んでいたら、pushする前にユーザーに一言確認する(意図しない上書きの可能性があるため)
2. 問題なければ以下を実行する
   ```bash
   git add -A
   git commit -m "<変更内容を要約した日本語の一言>"
   git push origin main
   ```
3. push完了後、次の3点をユーザーに伝える
   - どのフォルダ(サービス)を変更したか
   - Vercelへの反映には数分かかること
   - 反映を確認したいなら、そのサービスのVercel URLを一緒に開いて見た目を確認してよいこと

## 注意

- パスワードやSSHキーの入力を求められることはない(GitHubの認証は事前に設定済みのSSH鍵で自動的に通る想定)。もし認証エラーが出たら、SSH鍵がGitHubに登録されているか確認するようユーザーに伝えて、無理に進めない
- コミットメッセージは技術的な英語ではなく、ユーザーが読んでも分かる日本語の一言でよい(例:「投資家向けLPの見出しを修正」)
